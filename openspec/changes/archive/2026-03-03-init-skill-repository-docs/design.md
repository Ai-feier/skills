## Context

当前个人开发者已有一个基础的 skill 仓库，包含基本的目录结构说明和部分 skill。但缺少：
1. 统一的 `npx skills` 命令行工具使用说明
2. 完整的跨平台（OpenCode、Codex、Claude Code）安装指南
3. 规范的 skill 分类和整理

目标用户是希望安装和使用这些 skill 的开发者，需要清晰的安装步骤。

## Goals / Non-Goals

**Goals:**
- 提供完整的 OpenCode、Codex、Claude Code 三个平台的 skill 安装说明
- 说明 `npx skills` 命令的使用方式和参数
- 整理现有的 skill，形成规范的分类体系
- 创建易于维护和扩展的文档结构

**Non-Goals:**
- 不包含 skill 的开发教程（仅提供使用指南）
- 不实现新的 CLI 工具（仅说明现有工具用法）
- 不涉及服务器端部署或 CI/CD

## Decisions

### 1. 文档结构采用分层设计

**决策**: 使用 README + 独立文档页面的分层结构
- 主 README 提供快速入门和概览
- 独立文档页面提供详细平台安装指南

**备选考虑**:
- 单一 README 包含所有内容 → 内容过长，不易维护
- 全部放到 Wiki → 需要用户主动跳转

**理由**: 分层结构兼顾了快速入门和深度参考的需求

### 2. 安装方式以手动复制为主

**决策**: 文档以手动复制/symlink 为主要安装方式

**备选考虑**:
- 开发自定义安装脚本 → 增加维护成本
- 依赖特定包管理器 → 增加用户学习成本

**理由**: 三个平台的官方都推荐手动复制方式，简洁直接

### 3. skill 分类按功能领域划分

**决策**: 按功能领域对现有 skill 进行分类

**备选考虑**:
- 按平台划分 → 一个 skill 可能在多个平台通用
- 按字母顺序 → 缺乏语义关联

**理由**: 按功能领域更符合用户的使用习惯

## Risks / Trade-offs

- [风险] 用户可能不清楚自己的平台对应哪种安装方式 → 缓解：文档中明确区分三个平台
- [风险] `npx skills` 命令的具体用法可能因版本变化 → 缓解：说明通用用法并提供官方链接
- [风险] skill 目录结构可能与各平台最新规范不一致 → 缓解：基于各平台官方文档编写
