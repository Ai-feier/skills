# Lesson Skill — 铁律捕获

将踩坑经验和关键教训提炼为铁律，写入项目的 `MEMORY.md`。

## 概述

Lesson skill 帮助你在开发过程中自动记录重要的经验教训。它会将这些教训以结构化的"铁律"格式写入项目根目录的 `MEMORY.md` 文件，方便日后查阅，避免重复踩坑。

## 触发方式

### 手动触发
- 输入 `/lesson`
- 输入 `记录经验`
- 输入 `铁律`
- 输入 `记住这个`

### 自动检测
Agent 在以下场景会自动建议记录：
- 调试花费 2+ 次尝试
- 修正了一个错误的假设
- 发现了非显而易见的项目约定
- 遇到了反直觉的框架/工具行为
- 发现值得编码化的重复模式

## 前置条件

**重要**：Lesson skill 需要项目根目录存在 `MEMORY.md` 文件。

### 自动初始化（推荐）
Lesson skill 会在首次使用时自动检测并创建 MEMORY.md 文件（懒初始化）。

### 手动初始化
如果你想预先配置，可以在项目根目录创建 `MEMORY.md`：

```markdown
# MEMORY.md — 项目铁律

本文件记录了项目开发过程中总结的重要经验和规则。

## 开发规范
- (铁律将记录于此)

## 调试经验
- (调试经验将记录于此)

## 工具配置
- (工具配置经验将记录于此)

## 代码风格
- (代码风格相关规则将记录于此)
```

## 铁律格式

```markdown
- **{可执行的规则}** — 简短原因（最多15字）
```

### 示例

✅ 正确：
- **gitaly image tag 必须与 Chart.appVersion 一致** — 否则 CrashLoopBackOff
- **禁止在 CronJob 中使用 latest tag** — 导致不可复现部署

❌ 错误（太模糊）：
- 注意 Helm values 的配置 ← 不可执行
- Kubernetes 很复杂要小心 ← 不具体

## 使用示例

```
> /lesson

请输入要记录的铁律：
```

Agent 确认后会：
1. 读取现有的 MEMORY.md
2. 检查是否已有类似规则（去重）
3. 在合适的章节下追加新规则
4. 确认写入结果

## 去重逻辑

- **完全相同** → 跳过，告知用户"已存在类似铁律"
- **部分重叠** → 合并为更精确的版本
- **全新** → 追加到对应章节

## 章节分类

规则会自动归类到以下章节（如不存在则创建）：
- `## 开发规范` — 代码规范、架构约束
- `## 调试经验` — 问题排查、调试技巧
- `## 工具配置` — 工具使用、配置经验
- `## 代码风格` — 代码风格、命名规范
